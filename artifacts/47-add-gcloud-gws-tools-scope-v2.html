01

Problem & outcome

The server profile should provision the gcloud and gws command-line tools so they are available on the always-on development server.

The outcome is limited to installing the Homebrew formulae for the two tools. No Google Cloud, Google Workspace, authentication, account, shell, credential, or configuration setup is part of this work item.

02

Context

The captured source names the required Homebrew formulae: gcloud-cli for Google Cloud CLI and googleworkspace-cli for Google Workspace CLI.

Repository exploration found an existing roles/apps role described as server-only developer CLI tools installed via Homebrew. playbook-server.yml includes that role; playbook-thin.yml does not.

03

Decision tree

One entry per resolved branch. Each cites the source that resolved it.

Placement
Use the existing roles/apps role.

Resolved by repo pattern: the role is already the server-only Homebrew CLI bucket.

Profile reach
Server profile only.

Resolved by user confirmation: the tools must be available to playbook-server.yml and not to the thin client.

Config scope
Install binaries only.

Resolved by captured source and confirmation: do not add configs.

04

Alternatives considered

Why the chosen approach beat what was on the table.

Add a new role
Would isolate the two Google tools.
Rejected because the existing apps role is explicitly for loose server-only CLIs that do not warrant their own role.
Add to thin profile too
Would make the tools available on all Macs.
Rejected because the user explicitly requires server-only availability.
